Binding

Existen cuatro tipos de comunicacion entre el HTML y el componente.

  1. Interpolación - Comunicación desde el Componente al HTML
  2. Property binding - Comunicación desde el Componente al HTML
  3. Event binding - Cominicación desde el HTML al Componente
  4. Two-way binding - Comunicacion en doble dirección
Interpolación

Se utliza para mostrar un valor que tenemos en el componente en el template.

  • El valor se pone entre llaves {{valorParaMostrar}}
  • {{valorParaMostrar}}
Property binding

Se utiliza para asignar un valor a una propiedad de un elemento en el template.

  • La propiedad se pone entre corchetes y el valor de la propiedad es lo que se envia desde el Componente <button [disblaed]="valorDePropiedad">Botón</button>
Event binding

Se utiliza para enviar información desde el template al componente, normalmete para ejecutar una función en el componente.

  • El evento se pone entre parentesis y el valor es el codigo a ejecutar cuando <button (click)="accionEnComponente()">Botón</button>
Two-way binding

Se utiliza para mantener actualizado un valor en template y en el componente a la vez, si el valor cambia en el template cambia a la vez en el componente y viceversa.

  • Para este tipo de binding tenemos que usar la directiva ngModel entre corchetes y parentesis y el valor es la propiedad que vamos a utilizar
    <input type="text" [(ngModel)]="valorBidireccional">
  • {{valorBidireccional}}
